🕸️ 13. Knowledge Graph RAG(知识图谱)
■ 核心思想
不是简单做向量检索,而是从文档中提取实体和关系构建知识图谱。 检索时既走向量相似度,也走图关系路径,把多跳关联信息也找出来。
■ 架构
代码
文档 → 实体提取 → 关系提取 → 知识图谱(Neo4j / NetworkX)
↓
Query → 图查询 + 向量检索
↓
融合结果 → LLM
■ 代码示例(简化版 NetworkX 实现)
代码
1. 向量检索
2. 从 query 中提取实体
3. 图检索:找到相关实体的一跳邻居
4. 将图谱信息整合到向量检索结果中
